

# Content Page Design

## 目的

教材ページは学習コンテンツを提供するページとする。

利用者が教材を読み進め、理解し、次の教材へ進めることを目的とする。

---

## 基本方針

- 長文を読みやすくする
- 学習導線を明確にする
- サイドバーを活用する
- PC利用を前提とする
- スマートフォンにも対応する

---

## PCレイアウト

```text
Header（固定）
├ グローバルエリア
└ ページ固有エリア
  └ Breadcrumb / 教材メタ情報 / 完了ボタン（吹き出し誘導付き）

┌─────────────┬──────────────────┐
│ Sidebar     │ Content          │
│             │                  │
└─────────────┴──────────────────┘
```

---

## スマートフォンレイアウト

```text
Header（固定）
├ グローバルエリア
└ ページ固有エリア
  └ Breadcrumb / 教材メタ情報 / 完了ボタン（吹き出し誘導付き）

Sidebar (アコーディオン形式 / トグル)
Content
```

補足:

- Sidebarは `<details>` と `<summary>` を用いたアコーディオン（トグルカード）形式で表示する
- ページ読み込み時、画面幅が 720px 以下の場合は JavaScript により自動的に `open` 属性を削除してアコーディオンを「閉じた状態」にし、本文へのスクロールアクセスを邪魔しないようにする
- PC表示時（721px以上）はトグル動作を無効化し、トグルアイコンも非表示にして常に展開されたサイドバーとして動作させる

---

## Header

構成:

### グローバルエリア

表示内容:

- ロゴ
- このサイトについて
- 用語集
- 学習状況管理

### ページ固有エリア

教材ページでは以下を表示する。

表示内容:

- Breadcrumb (パンくずリスト。左端に表示)
- 完了ボタン (ヘッダーの右端に固定配置。吹き出し誘導付き)

補足:

- 教材ページ固有の操作を集約する
- 余分なバッジ部品（難易度、推奨プラン、利用サービス等のメタ情報）は、表示上のノイズを減らすため廃止された。
- 完了ボタンはCSSで `margin-left: auto;` を指定し、常にヘッダーの右端に綺麗に固定されて配置される。
- 共通Headerを維持しながら教材UXを向上させる

---

## Sidebar

目的:

- カテゴリ内の教材一覧表示
- 現在位置表示
- 学習導線提供

表示仕様:

- **通常折りたたみとフローティング開閉バッジ（Gemini風）**:
  - 通常時は閉じられた状態を標準とし、固定ヘッダー直下の左上に丸型の「開閉用フローティングバッジ（`☰` アイコン）」を浮遊配置する。
  - バッジをクリックしてアコーディオンが展開された際、リスト本体はブランドカラーを意識した極薄ティール色（`var(--color-accent-muted)`：`#e6f0ef`）を背景とする、影・ボーダー・角丸付きの美しい「独立したカード部品」としてポップオーバー（浮き出し）表示される。
  - 閉じているときは `☰`（ハンバーガーアイコン）、展開しているときは `✕`（閉じるアイコン）にバッジ内の表示が自動でトグルされ、マウスオーバー時にはそれぞれ「サイドバーを開く」「サイドバーを閉じる」というティール半透明の吹き出しツールチップがスムーズなスライドアニメーション付きで動的出現する。
- **完了ボタンとの自動展開連動**:
  - 詳細ページの完了ボタンをクリックしたタイミングで、サイドバーアコーディオンが自動的に連動して展開（`open`）される。
- **学習中の現在位置と完了状態の可視化**:
  - 現在学習中のカレント教材の左側には `👉` アイコンが心地よいパルスアニメーション付きで表示される。
  - 完了した教材の右側には、LocalStorage上の進捗を動的に検知してインラインで `✅` バッジが表示され、未実施の教材は非表示とする。

## Content

表示内容:

- 教材タイトル
- 本文
- Alert
- Tip
- Exercise
- Code Block
- Table

補足:

- 本文を主役とし、装飾より可読性を最優先する。
- サイドバーがフローティングバッジ化されたことに伴い、従来の横並びグリッドレイアウトを廃止し、本文エリアを最大幅（`760px`）まで横いっぱいに広げた単一カラム中央寄せレイアウトへと刷新した。

---

## 完了状態管理

目的:

- 学習完了記録

方針:

- 教材ページから完了状態を変更可能とする

補足:

- 完了ボタンはHeaderに配置する
- 教材ページのどこからでも操作できるようにする
- 完了状態は即時保存する

---

## 完了誘導

目的:

- 完了ボタンの押し忘れ防止

方針:

- ページ最下部到達時に、固定ヘッダーの完了ボタンから半透明の吹き出しで誘導を表示する。

表示仕様:

- ユーザーが未完了の状態でページ最下部（底から 30px 以内）にスクロールしたことを JavaScript で検知すると、固定ヘッダーの「完了にする」ボタンの直下に「読み終わったら完了を押しましょう。」という吹き出し（`.completion-tooltip`）を表示する。
- 吹き出しは、全体カラー（アクセント色のティール `#2f6f73`）を意識した美しい半透明背景（`rgba(47, 111, 115, 0.88)`）に、背景ぼかし（`backdrop-filter: blur(8px)`）を効かせたグラスモーフィズムデザイン。
- ボタン側に向く小さな上向きの矢印（ツノ）を配し、表示開始時には上からスライドしながらふわっと浮かび上がる微細なアニメーションを伴う。
- 吹き出しは表示後 **3秒間** 経過すると自動でフェードアウトして非表示になる。
- 既に完了状態の教材である場合、または最下部到達前にユーザー自身で「完了にする」をクリックした場合は、スクロール検知や吹き出し表示を行わない安全な制御。完了ボタンクリック時には、吹き出しが表示中であれば即時に非表示へトグルされる。

---

## Previous / Next Navigation

表示方針:

- ページの極限までのシンプル化と教材本文への集中を優先するため、ページ最下部のナビゲーションカード（前後の教材へのリンクカード）は**廃止**とする。
- カテゴリ内の別教材への移動は、PC・スマホともにサイドバーから一元的に行う構成とする。

---

## 教材メタ情報

表示方針:

- ページの極限までのシンプル化と学習集中を優先するため、詳細ページの固定ヘッダー部分での「教材メタ情報（難易度、プラン、サービス等）」の表示は**廃止**（非表示）とする。
- ただし、各教材のメタデータ（YAML）としては内部的に管理・保持し、完了状態のトラッキング等のために活用する。
