

# Content Page Design

## 目的

教材ページは学習コンテンツを提供するページとする。

利用者が教材を読み進め、理解し、次の教材へ進めることを目的とする。

---

## 基本方針

- 長文を読みやすくする
- 学習導線を明確にする
- サイドバーを活用する
- PC利用を前提とする
- スマートフォンにも対応する

---

## PCレイアウト

```text
Header（固定）
├ グローバルエリア
└ ページ固有エリア
  └ Breadcrumb / 教材メタ情報 / 完了ボタン

┌─────────────┬──────────────────┐
│ Sidebar     │ Content          │
│             │                  │
└─────────────┴──────────────────┘

Previous / Next
```

---

## スマートフォンレイアウト

```text
Header（固定）
├ グローバルエリア
└ ページ固有エリア
  └ Breadcrumb / 教材メタ情報 / 完了ボタン

Content

Previous / Next
```

補足:

- SidebarはDrawer表示する
- ハンバーガーメニューからDrawerを利用する

---

## Header

構成:

### グローバルエリア

表示内容:

- ロゴ
- このサイトについて
- 用語集
- 学習状況管理

### ページ固有エリア

教材ページでは以下を表示する。

表示内容:

- Breadcrumb
- 教材メタ情報
- 完了ボタン

補足:

- 教材ページ固有の操作を集約する
- 共通Headerを維持しながら教材UXを向上させる

---

## Sidebar

目的:

- カテゴリ内の教材一覧表示
- 現在位置表示
- 学習導線提供

表示内容:

- カテゴリ名
- 教材一覧
- 完了状態

補足:

- ページ内目次は表示しない
- 教材間移動を優先する

---

## Content

表示内容:

- 教材タイトル
- 本文
- Alert
- Tip
- Exercise
- Code Block
- Table

補足:

- 本文を主役とする
- 装飾より可読性を優先する

---

## 完了状態管理

目的:

- 学習完了記録

方針:

- 教材ページから完了状態を変更可能とする

補足:

- 完了ボタンはHeaderに配置する
- 教材ページのどこからでも操作できるようにする
- 完了状態は即時保存する

---

## 完了誘導

目的:

- 完了ボタンの押し忘れ防止

方針:

- ページ最下部到達時に完了誘導を表示する

表示例:

```text
✓ 読み終わりましたか？

[完了にする]
```

補足:

- 半透明フローティング表示を想定する
- 強制表示は行わない
- 完了ボタンの代替ではなく補助機能とする

---

## Previous / Next Navigation

目的:

- 学習順移動

表示内容:

- 前の教材
- 次の教材

補足:

- 学習導線として重要な要素とする

---

## 教材メタ情報

表示内容:

- 難易度
- 推奨プラン
- 利用サービス

例:

```text
ビギナー
ChatGPT
Free
```
