# Category Page Design

## 目的

カテゴリページは学習カテゴリの入口とする。

利用者がカテゴリの概要を理解し、学習したい教材を選択できることを目的とする。

---

## 基本方針

- カテゴリ概要を最初に表示する
- 教材一覧を主役とする
- Heroは採用しない
- 一覧性を重視したリスト型カードを採用する

---

## 構成

```text
Header

Breadcrumb

Category Overview

Learning Card
Learning Card
Learning Card
...
```

---

## Breadcrumb

目的:

- 現在位置表示
- 上位階層への移動

例:

```text
読んで学ぶ > 基礎
```

---

## Category Overview

目的:

- カテゴリ概要説明

表示内容:

- カテゴリ名
- カテゴリ説明
- 推奨対象
- 学習目安
- 対応プラン目安

補足:

- Heroコンポーネントは利用しない
- シンプルな概要表示とする

---

## 教材一覧

目的:

- 学習教材選択

表示方法:

- Learning Cardを利用する
- アイコン＋説明型のレイアウトを採用する

表示内容:

- 教材タイトル
- 教材概要
- 難易度
- 利用サービス
- 推奨プラン
- 完了状態

補足:

- カテゴリ内の教材を一覧表示する
- 完了済み教材は視覚的に区別する
- レベル別グルーピングを表示できるようにする

---

## Learning Card

レイアウト:

```text
┌────┬─────────────────────────────────┐
│Icon│ タイトル                        │
│    │ 概要説明                        │
│    │ [難易度] [プラン] [サービス]   │
│    │ [完了]                          │
└────┴─────────────────────────────────┘
```

表示内容:

- アイコン
- タイトル
- 概要
- 難易度
- 推奨プラン
- 利用サービス
- 完了状態

動作:

- アイコンまたはカード全体から教材ページへ遷移する

補足:

- Google Sitesの一覧に近いイメージとする
- カードよりも教材リストとしての視認性を優先する
- ハンズオン教材では利用サービスの表示を重視する
- サービス依存の教材を判別しやすくする
- レイアウト図は概念図とする
- 実装時は可読性・レスポンシブ対応・デザイン方針を優先する
- 図の完全再現を目的としない
- 本設計で重要なのは一覧性と情報構造である
- アイコン・タイトル・概要・メタ情報を分かりやすく配置できればレイアウト調整を許容する
---

## 教材メタ情報

教材ごとに以下の情報を保持する。

- タイトル
- 概要
- 難易度
- 推奨プラン
- 利用サービス
- 完了状態

利用サービス例:

- ChatGPT
- Gemini
- Claude
- NotebookLM
- Cursor

補足:

- ハンズオン教材では利用サービスが重要な情報となる
- 音楽生成や動画生成など、特定サービス前提の教材を想定する
- サービス名の記載は許容する


---

## レベル別グルーピング（案）

目的:

- 学習順序を明確にする
- 利用可能なプランの目安を示す

例:

```text
ビギナー
──────────
📘 AIの歴史
📘 AIの種類

スタンダード
──────────
📘 プロンプトエンジニアリング
📘 カスタムGPT

アドバンスド
──────────
📘 RAG
📘 MCP
📘 AIエージェント
```

補足:

- ビギナーはFreeプランでも学習可能な内容を中心とする
- スタンダード以降は有料プラン推奨の内容を含む
- 特定サービスに依存しない表現を基本とする
- 必要に応じて「ChatGPTならPlus以上推奨」などの補足を行う
