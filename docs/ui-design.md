# UI Design

## 目的

本ファイルはUI設計方針を管理する。

レイアウト詳細やコンポーネント詳細ではなく、サイト全体のUI設計思想を記録する。

---

## 対象デバイス

優先対象:

- デスクトップPC
- ノートPC

対応対象:

- タブレット
- スマートフォン

---

## 基本方針

- PCファーストで設計する
- レスポンシブ対応を行う
- スマートフォンでの閲覧を可能とする
- スマートフォンでの学習も考慮する
- ただし全コンテンツのスマートフォン完遂は前提としない

例:

- AIコーディング
- MCP
- RAG

などはPC利用を前提としてもよい。

---

## デザイン方針

参考イメージ:

- GitBook系の情報設計
- Notion系の読みやすさ
- サントリーサイトの柔らかい雰囲気

方針:

- 読みやすさを最優先とする
- 派手さより理解しやすさを重視する
- 学習継続しやすいUIを目指す
- 長文コンテンツの可読性を重視する
- カテゴリページはカードUIを活用する

配色方針:

- ライトテーマを採用する
- 純白ではなく少し柔らかい背景色を利用する
- 強いブランドカラーへの依存を避ける
- 学習の邪魔にならない配色を優先する

避けるもの:

- 過度なアニメーション
- 強すぎるグラデーション
- ダークテーマ前提の設計
- 装飾過多なUI
