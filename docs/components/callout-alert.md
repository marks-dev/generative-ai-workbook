# Component Design: Callout Alert (ポップオーバー型アラート)

## 実装仕様 (Implementation Specification)
- **実装パターン**: パターン3（JS動的生成 ＋ 共通CSS）
- **主要関連ファイル**:
  - **JavaScript (動的生成・配置制御)**: [src/assets/js/learning-progress.js](../../src/assets/js/learning-progress.js)
  - **CSS (スタイル)**: [src/assets/css/site.css](../../src/assets/css/site.css)

## 用途
コールアウトアラート（Callout Alert）は、教材ドキュメント内の補足や注意喚起など（執筆時に Markdown の `blockquote` と `[!TYPE]` で書かれたもの）を、本文の通読の流れを妨げないように、見出し横の小さなアイコンとホバー吹き出し（ツールチップ）の組み合わせに変換して提供するUIコンポーネントです。

## 利用画面
- 教材詳細ページ（本文エリア内）
- 「このサイトについて」ページなど、Markdown から変換される全ページ

## アラートタイプとビジュアル定義
アラートタイプごとに、対応するアイコン（SVG）とテーマカラーが割り当てられ、意味合いを視覚的に伝えます。
1.  **`[!NOTE]` (補足 / ℹ️)**: ティール・ブルー系のカラー。一般的な背景知識や管理者への報告注記等に使用。
2.  **`[!TIP]` (コツ / 💡)**: グリーン・イエロー系のカラー。効率的な操作や実務でのTipsに使用。
3.  **`[!IMPORTANT]` (重要 / 📢)**: オレンジ・ゴールド系のカラー。動作要件や必須要件に使用。
4.  **`[!WARNING]` (注意 / ⚠️)**: オレンジ・レッド系のカラー。エラー回避や仕様変更などの注意喚起に使用。
5.  **`[!CAUTION]` (警告 / 🚨)**: レッド系のカラー。データの消失やセキュリティリスク等の重大な警告に使用。

## 動作・配置（動的レイアウト）仕様
- **クライアントサイド変換**:
  - HTMLロード時、JSが `<blockquote>` タグを走査し、`[!TYPE]` 記号を検知すると、それをトリガーバッジ（`.alert-trigger`）とツールチップ（`.alert-tooltip`）の構造へ動的に置換します。元の `<blockquote>` は非表示（`display: none`）になります。
- **自動挿入位置の特定（親要素の遡及判定）**:
  - JSが対象のアラート要素からDOMツリーを上方向に遡り、最も近い見出し要素（`h2`, `h3`, `h4`）を自動検出します。
  - 見出しが見つかった場合、その見出しテキストの末尾にインライン要素としてバッジをアペンドします。
  - 見出しが見つからない場合は、元の位置にスタンドアロンバッジ（`.alert-trigger--standalone`）として配置します。
- **衝突検知（スマート・ポジショニング）**:
  - デスクトップ表示時、トリガーのホバー/フォーカスイベント発生時にJSが位置計算を行います。
  - トリガーの上部スペースと固定ヘッダー（`site-header` + `context-header` = 計108px）の高さを比較し、上部にツールチップを表示する十分な余白がない場合は、動的に下向きの吹き出しスタイル（`.alert-tooltip--bottom`）を適用します。
- **アクセシビリティ・モバイル対応**:
  - 各トリガーバッジには `tabindex="0"` および `role="button"` を付与し、キーボードの `Tab` 移動およびフォーカス操作でツールチップが表示できるようにします。
  - モバイル環境（画面幅 720px 以下）では、衝突検知をスキップし、CSSメディアクエリによって本文内に静的カード形式、もしくは最適な絶対配置でツールチップが潰れずに表示されるようにレスポンシブスタイルを定義します。
