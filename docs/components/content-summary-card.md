# Component Design: Content Summary Card (教材まとめカード)

## 実装仕様 (Implementation Specification)
- **実装パターン**: パターン1（Nunjucks静的定義 ＋ 共通CSS）
- **主要関連ファイル**:
  - **HTML (テンプレート)**: [src/_includes/content.njk](../../src/_includes/content.njk)
  - **CSS (スタイル)**: [src/assets/css/site.css](../../src/assets/css/site.css)

## 用途
教材まとめカード（Content Summary Card）は、教材詳細ページの最下部（フッター領域）に配置され、その教材で学んだことの要点（抑えるべきポイント）と、次に進むべき関連教材（座学 ⇄ 実践の架け橋）を一つのカード型UIブロックに統合して提供するコンポーネントです。

## 利用画面
- 教材詳細ページ（メインコンテンツ最下部）

## 構成・データ連動
Nunjucksの Front Matter に `keypoints` または `relatedHandson` が指定されている場合にのみ、ページの末尾に自動的にレンダリングされます。

カードの内部は以下の2つのセクションから構成されます。

### 1. 抑えるべきポイント (Keypoints)
*   **用途**: 教材の最も重要な概念や行動指針を3〜4項目の箇条書きで整理します。
*   **データ**: Front Matter の `keypoints` 配列から動的にループ生成されます。
*   **ビジュアル**: タイトル横のSVG同心円アイコンと、見やすいインライン余白を持つリストで構成されます。

### 2. 関連コンテンツリンク (Related Content Area)
*   **用途**: 現在閲覧している教材の対になる教材（座学「読んで学ぶ」なら対応するハンズオン「触って学ぶ」、逆もまた然り）へのナビゲーションリンクを表示します。
*   **データ**: Front Matter の `relatedHandson` 配列から動的に生成されます。
*   **詳細仕様**: リンクカード自体のレイアウトや、ディレクトリ判定による「関連ハンズオン/関連読み物」の見出し・絵文字の動的切替挙動の詳細は、[related-content-card.md](related-content-card.md) を参照してください。

## デザイン・スタイル
- ページ全体の本文と明確に区別するため、カード全体には微細なドロップシャドウ、丸角、および柔らかい境界線（ボーダー）が適用されています。
- PC表示では、2つのセクション（KeypointsとRelated Links）がフレキシブルにレイアウトされ、モバイル環境（画面幅 720px 以下）では自動的に縦積みの1カラムレイアウトへ崩れないように折り返されます。
