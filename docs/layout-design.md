# Layout Design

## 目的

本ファイルは画面レイアウト設計を管理する。

コンポーネントの見た目ではなく、各画面にどの領域を配置するかを定義する。

---

## 共通レイアウト

PC:

```text
┌──────────────────────────────┐
│ Header                       │
├──────────────────────────────┤
│ Main Content                 │
└──────────────────────────────┘
```

スマートフォン:

```text
┌──────────────────────────────┐
│ Header                       │
├──────────────────────────────┤
│ Main Content                 │
└──────────────────────────────┘
```

---

## トップページ

目的:

- サイトの入口
- 学習導線の案内
- 続きから学習

構成:

```text
Header

学習の進め方

┌─────────────────┬─────────────────┐
│ 読んで学ぶ      │ 触って学ぶ      │
└─────────────────┴─────────────────┘

学習状況サマリー

続きから学習
```

補足:

- サイドバーは表示しない
- Heroは採用しない
- 「このサイトについて」はHeaderから遷移する
- 「読んで学ぶ」と「触って学ぶ」は同列で扱う

---

## カテゴリページ

目的:

- カテゴリ概要表示
- 教材一覧表示

構成:

```text

Breadcrumb

Content Card
```

補足:

- カード一覧を中心とする

## 教材ページ

目的:

- 学習コンテンツ表示

PC構成:

```text
Header

Breadcrumb

┌─────────────┬──────────────────┐
│ Sidebar     │ Content          │
│             │                  │
└─────────────┴──────────────────┘

Related Content

Previous / Next
```

スマートフォン構成:

```text
Header

Breadcrumb

Content

Related Content

Previous / Next
```

補足:

- サイドバーは教材ページのみ表示する

---

## Header

方針:

- ロゴをトップページへの遷移部品として利用する
- 専用の「ホーム」メニューは作成しない

- Headerは固定表示（Sticky Header）とする
- スクロール時も常にアクセス可能とする

主要メニュー:

- このサイトについて
- 用語集
- 学習状況管理

補足:

- ロゴからトップページへ遷移する
- 「読んで学ぶ」「触って学ぶ」「学習の進め方」への導線はトップページに配置する
- スマートフォンではハンバーガーメニューへ格納する

- 長文教材でも主要導線へ即座にアクセスできるようにする
- 学習状況管理へのアクセス性を維持する
- ライセンス情報や著作権表記は「このサイトについて」に集約する
- 作成者表記は「このサイトについて」に記載する
- 作成者名は「MARKS AI Lab」とする
- Footerは採用しない

---

## 学習状況管理

方針:

- 独立ページは作成しない
- Headerからアクセスできるようにする
- エクスポート機能を提供する
- インポート機能を提供する

補足:

- 学習状況保存方式の説明は「学習の進め方」に記載する
- LocalStorage利用について説明する
- JSONエクスポート・インポートについて説明する
