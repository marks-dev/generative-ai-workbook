

# Learning Progress Design

## 目的

本ファイルは学習状況管理機能の設計を管理する。

利用者が学習の進捗を把握し、途中から再開できることを目的とする。

---

## 基本方針

- サーバーを利用しない
- GitHub Pagesのみで動作する
- 学習履歴はブラウザ内へ保存する
- 利用者登録は行わない
- 個人情報は保存しない

---

## 保存・入出力方式

### 保存方式
- LocalStorageを利用してブラウザ内に保存する。
- キー名: `generative-ai-workbook.learningProgress.v1`

### データ入出力（エクスポート・インポート）
- 学習状況をJSON形式でローカルファイルにエクスポート、およびインポートできる。
- **データスキーマ**:
  ```json
  {
    "version": 1,
    "completed": {
      "<completion-id>": {
        "completed": true,
        "title": "<教材タイトル>",
        "updatedAt": "<ISO 8601形式の日時>"
      }
    }
  }
  ```
  - `completion-id` は、教材URLからドメインおよび共通プレフィックスを除去し、スラッシュをハイフンに変換した一意の識別子。

### リセット機能
- 学習状況管理UI（ダイアログ）内に「全状況をリセット」ボタンを配置する。
- 押下時に確認ダイアログを表示し、ユーザーが承認した場合はLocalStorageのデータをクリア（空の `completed` 状態）にし、同一画面内のすべての進捗バッジや完了ボタン等の表示を即座にリセット（未実施状態に同期）する。

---

## 管理単位

学習状況は教材単位で管理する。

原則:

- 1教材 = 1進捗データ

例:

- AIの歴史
- プロンプト入門
- RAG

---

## 学習状態

各教材は以下の状態を持つ。

- 未実施
- 完了

補足:

- 初期実装では状態を最小限にする
- 上記以外の状態は管理しない
- 将来必要になった場合のみ追加を検討する

---

## 表示ルール

トップページ:

- 学習状況サマリーは表示しない（極限のシンプルさと学習集中を優先するため）。ダイアログ内での進捗件数表示のみとする。

カテゴリページ（サブカテゴリ・ポータル）:

- 各教材、および子カテゴリの完了状態を表示する。
- 完了したカードは、右端中央に `✅` バッジを表示し、カード全体を subtle なグリーン調のデザイン（`.is-completed`）に切り替える。

教材ページ:

- 現在状態を表示する
- 状態変更（完了チェックのトグル）を可能とする
- 未完了状態の際、スクロールがページ最下部（底から 30px 以内）に到達したことを検知すると、固定ヘッダーの完了ボタン直下に、全体カラー（ティール）を意識した半透明の「完了誘導吹き出し」（3秒自動フェードアウト付き）を動的に表示して学習完了を促す。

サイドバー:

- 各教材の学習完了状態を動的に表示する。
- 完了した教材の右端には `✅` バッジを表示し、未実施の教材は何も表示しない。
- 固定ヘッダーの「完了にする」ボタン押下（完了にした瞬間）に連動して、サイドバーアコーディオンを自動的に展開（`open`）させる。

---

## 完了判定

詳細教材完了:

- 利用者が教材ページで「完了」としてマークし、LocalStorage に `completed_${id}` が保存された状態。
- カテゴリページでは、各教材カードに定義された `data-completion-id` 属性を元に LocalStorage を参照し、完了していれば `✅` バッジを表示する。
- 教材詳細ページのサイドバーでは、静的なマークアップに依存しない堅牢な動的同期を行うため、HTML の `data-completion-status-url` 属性に指定された各教材の `url` から JavaScript が自動的に一意の `completionId`（パスプレフィックスを除去しスラッシュをハイフンに置換する規則）を安全に抽出・パースし、LocalStorage の完了進捗を検知して `✅` を動的表示する。

親カテゴリ（ポータル・サブカテゴリ）完了:

- 配下に紐づくすべての詳細教材が完了している状態。
- カテゴリページの親カードに定義された `data-completion-ids`（カンマ区切りの ID リスト）属性を元に JavaScript で LocalStorage を走査し、リスト内のすべての ID が「完了」状態である場合のみ、親カードも「完了（`✅` 表示＋グリーン調の背景トグル）」と判定する。

学習ルート完了:

- 配下カテゴリが全て完了している状態
- 学習再開支援機能（直前に完了した教材の次の教材をレコメンドする等）は、本サイトの自由な学習導線コンセプトに基づき、初期実装および現時点では非対応とする。

補足:

- 初期実装では進捗率（%）を管理しない
- チェックリスト方式を採用する
- 完了状態を積み上げて進捗を表現する
