# Component Design

## 目的と基本方針
本ファイルは、各画面で再利用するUI部品（共通コンポーネント）の設計を管理し、デザインおよび実装の一貫性を保つことを目的とします。
- **基本方針**: 共通化による学習体験の一貫性確保、過度なコンポーネント分割の回避、可読性の最優先。

---

## レイアウト系コンポーネント
### Header
- **用途**: グローバル・コンテキストナビゲーションおよび教材固有操作の表示
- **構成**: Global Area（ロゴ、このサイトについて、用語集、学習状況管理）、Context Area（パンくずリスト、完了ボタン）
- **利用画面**: 全ページ

### Sidebar
- **用途**: カテゴリ内教材一覧、現在位置、および各教材の学習完了状態（✅）の表示
- **動作仕様**: PC表示時は常時展開、スマートフォン表示時はトグル開閉するフローティングカード形式。
- **利用画面**: 教材詳細ページ

---

## ナビゲーション系コンポーネント
### Breadcrumb
- **用途**: 現在位置の階層表示および上位ページへの遷移（Header Context Area内に配置）
- **利用画面**: カテゴリページ、教材詳細ページ

### Complete Button
- **用途**: 教材の学習完了状態の変更（LocalStorageへの即時保存およびサイドバーとの連動トグル）
- **利用画面**: 教材詳細ページ（Header Context Area右端に固定配置）

### Hamburger Menu
- **用途**: スマートフォン表示時にサイドバーを開閉するためのフローティングボタン

---

## コンテンツ系コンポーネント
### Category Overview
- **用途**: カテゴリ概要のスクリーンリーダー用マークアップ（`h1` を `visually-hidden` で保持し、画面上は非表示）
- **利用画面**: カテゴリページ

### Learning Card
- **用途**: カテゴリページにおける教材一覧の表示（カード型リストUI）
- **構成**: 左端アイコン、中央タイトルと説明文、右端中央完了バッジ（✅）の3カラム構成。
- **利用画面**: カテゴリページ、トップページ（ルート入口タイル）
- **補足**: 完了時は subtle なグリーン枠線と淡いグリーン背景（`.is-completed`）にトグルする。親カテゴリカードは全子教材の完了時に ✅ を表示する。

### Navigation Tile
- **用途**: トップページにおける学習ルート入口（「読んで学ぶ」「触って学ぶ」）の表示
- **動作仕様**: カード全体をリンクとし、hover/focus時にアクセントカラーで強調。内部に別ボタンや説明文は持たない。
- **利用画面**: トップページ

### Related Content Card
- **用途**: 教材詳細ページ最下部のまとめカード（スタック形式）内の、関連コンテンツへの相互リンク表示
- **動作仕様**: `src/reading` 配下なら「関連ハンズオン（🛠️）」、`src/hands-on` 配下なら「関連読み物（📖）」と動的に切り替える。
- **利用画面**: 教材詳細ページ

## 学習状況系コンポーネント
### Completion Badge
- **用途**: 学習状況のインライン表示（未実施：非表示 / 完了：`✅`）
- **利用画面**: カテゴリページのカード、および教材詳細ページのサイドバー

### Completion Prompt (誘導ツールチップ)
- **用途**: 未完了状態でページ最下部に到達した際、ヘッダーの完了ボタン直下に出現する吹き出し誘導UI（3秒自動フェードアウト）
- **デザイン**: ティール半透明背景（`rgba(47, 111, 115, 0.88)`）と背景ぼかし（backdrop-filter）のグラスモーフィズム。

---

## コンテンツ装飾系（独自拡張Markdownレンダラー）
- **Alert**: 注意事項やハルシネーション警告を表示（Notion風 `[!IMPORTANT]` / `[!WARNING]` 構文）
- **Tip**: 補足知識や実務ノウハウを表示（Notion風 `[!TIP]` / `[!NOTE]` 構文）
- **Exercise**: 理解確認や実践課題の表示（ハンズオンや練習問題用）
- **Code Block**: コードやコマンドの等幅表示。右上に動的に「📋 コピー」ボタンを配備し、コードと重ならない余白（padding-top）を確保。
- **Table**: データの比較や一覧表示。
