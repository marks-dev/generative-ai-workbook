# CSS Design

## 目的

サイト全体のスタイル方針を定義する。

学習コンテンツの読みやすさを最優先とする。

---

## 基本方針

- 可読性を最優先とする
- 装飾より情報伝達を優先する
- 長期保守しやすい構成とする
- レスポンシブ対応を前提とする
- AIが理解しやすい構成とする

---

## CSS管理方針

方針:

- CSSは共通資産として管理する
- 実装時にページごとに生成しない
- AIに都度CSS生成をさせない
- 共通CSSを読み込む方式を採用する

目的:

- デザインの一貫性維持
- AI生成差異の排除
- 保守性向上

補足:

- CSSは原則として一度設計し共通利用する
- コンテンツとスタイルを分離する

---

## デザインコンセプト

参考イメージ:

- 三井不動産
- サントリー

特徴:

- 落ち着いた印象
- 過度な装飾を行わない

---

## タイポグラフィ

方針:

- 読みやすさ重視
- 日本語表示を優先
- インラインおよび見出しレベルで視覚的なメリハリを定義する

見出し階層ルール:

- **H1 (大見出し)**: ページタイトル。アクセントカラーの下線（`2px`）を引くことで、大タイトルとしての佇まいを引き締める。先頭に自習用ワークブックを象徴する共通グリフィコン（📖）を定義し、テキストと縦中央で美しく整列させる。
- **H2 (中見出し)**: 左側に `::before` 擬似要素によるアクセントカラーの縦長バー（幅6px、角丸）をPrefixマークとして配置し、見出しテキストと縦中央で整列させる。既存の下線は廃止する。
- **H3 (小見出し)**: 左側に細い左ボーダー（4px）をあて、左余白（インデント）を設ける。
- ※ これにより、`H1`（下線）→ `H2`（太バーPrefix）→ `H3`（左細ボーダー）という、非常に美しく整然とした Notion風 のインライン階層を形成する。

インライン要素ルール:

- **太字 (`strong`)**: フォントウェイトを確実に太く（`700`）定義し、純粋なテキスト色でインラインでの力強いメリハリを確保する。
- **インラインコード (`code`)**: 背景色に加え、文字色をアクセントカラー（`var(--color-accent)`）に変更し、地の文との境界を明快に際立たせる。フォントサイズも `0.9em` に定義する。
