# UI Design (サイト共通UI・コンテンツ記述仕様)

## 目的と基本方針
本ファイルは、サイト全体のUI設計思想、共通の表示/インタラクションルール、教材コンテンツの共通記述ルール（Markdownガイドライン）、およびメディアアセット掲載基準を定義し、人間による執筆、AIによる自動生成、およびWebアプリケーションの実装における品質と一貫性を保つことを目的とします。
- **Markdown正本主義**: 原則としてコンテンツはMarkdownで管理し、HTMLは生成物とします。人間が直接修正しやすく、AIが生成しやすい構成を優先します。
- **標準Markdown優先**: 標準Markdownを中心として運用し、独自記法は極力導入しません。
- **可読性優先**: GitBook系の情報設計、Notion系の読みやすさを目指し、まぶしさを抑えたライトテーマの柔らかい背景色を採用します。

## 対象デバイスと共通レイアウト構造

### 1. 対象デバイスと基本方針
- **優先対象**: デスクトップPC、ノートPC
- **対応対象**: タブレット、スマートフォン（※スマートフォンでの学習も考慮しますが、全コンテンツのスマートフォン完遂は前提としません。PC利用を推奨します。）

### 2. 共通レイアウト構造（スケルトン）
サイト共通の基本骨格として、HeaderおよびMain Contentを配置する縦2段構造を基本とします。詳細は [header.md](../components/header.md) および各個別画面設計書（`docs/pages/*.md`）を参照。
- 本文エリア（`.content-article`）は最大幅 `1040px` (`var(--container-width)`) まで拡張された中央寄せのレイアウトです。

## 外部リンクの取り扱い (共通ルール)
現在のドメインと異なる外部サイトへのリンク（MIT Licenseや参考ドキュメントへのリンクなど）の挙動方針を定義します。

- **方針**: 外部リンクは、現在の学習セッション（コンテキスト）を維持するため、常にブラウザの別タブで開きます（`target="_blank"`）。
- **セキュリティ**: セキュリティ保護のため、必ず `rel="noopener noreferrer"` を付与します。
- **自動化実装**: 共通の JavaScript にて、ページロード時に `http` または `https` から始まるすべてのリンクを自動走査し、現在のホスト名と異なる場合にこれらの属性を付与します。これにより、手動での属性付与漏れを防ぎ一貫性を保証します。
- **記述ルール (執筆者向け)**: 執筆時（Markdown上）は、標準のMarkdownリンク記法 `[アンカーテキスト](URL)` で記述します。セキュリティ属性（`rel`）や別タブ表示の指定はシステム側で自動付与するため、Markdown内にHTMLタグなどを手動で記述する必要はありません。

## 共通の記述・装飾系UI仕様 (Markdown拡張)
Markdownからレンダリングされる要素、またはカスタムHTML要素に適用される共通の装飾UI設計および執筆ルールです。

### 1. 見出し階層
- **記述ルール**: `## 大項目` ＞ `### 中項目` ＞ `#### 小項目` の最大4階層とし、深くしすぎない。

### 2. Alert / Tip (Notion風のアラートボックス)
- **Alert**: 注意事項やハルシネーション警告には `[!IMPORTANT]` / `[!WARNING]` 構文を使用します。
- **Tip / Note**: 補足知識や実務ノウハウには `[!TIP]` / `[!NOTE]` 構文を使用します。
- **UI表現**: 強い警告色ではなく、Notion風の背景とアイコンで表現。視覚的な囲み枠、十分なパディング、および左端の識別用アイコンでレイアウトを際立たせます。

### 3. Exercise (理解確認・実践課題)
- **記述ルール**: 理解確認や実践課題を掲載する際は、専用の章または記述ブロックとして定義します。
- **UI表現**: 通常の本文やAlertと区別できる専用の囲み枠、タイポグラフィ（タイトル＋タスク指示）を定義します。

### 4. Code Block (等幅コード表示)
- **記述ルール**: ソースコードやコマンドは、標準のバッククォーツ3つで囲み、必ず言語名を指定します。
- **UI表現**:
  - **動的コピーボタン**: JavaScriptにより、全コードブロックの右上に動的に「📋 コピー」ボタンを配備します。
  - **レイアウト配慮**: ボタンとソースコードのテキストが重ならないよう、コードブロック上部に適切な余白（padding-top）を確保する共通スタイルを適用します。

### 5. Table (データ比較)
- **記述ルール**: データの比較や一覧表示において、表形式が明らかに分かりやすい場合のみ利用します。
- **UI表現**: 罫線の太さを極力抑え、交互の行の背景色（縞模様）やヘッダーの強調スタイルなど、Notion風のシンプルで視認性の高いテーブルデザインを適用します。

## メディア・アセット掲載ガイドライン
教材ページや共通UIに画像、動画、アイコンなどのアセットを掲載する場合は、以下のルールを遵守します。

### 1. 画像・アイコンアセット (WebP形式に完全統一)
- **原則**: 画像やアイコンなどの画像アセットは**WebP形式（`.webp`）のみ**とし、`src/assets/images/` （および共通アセットの `assets/` 配下）に格納し、相対パスで呼び出します。
- **AI生成動作**: AIが自動生成する際、ライセンスフリーの画像が利用可能な場合は、取得・WebP変換・格納・Markdown埋め込みを自動で実行します。

### 2. YouTube動画 (埋め込み禁止・注記付き外部リンク化)
- **原則**: 直接の iframe 埋め込みは一切禁止（将来のリンク切れやCORS警告を防止）。
- **掲載方式**: 必ず標準のリンクとして記述し、直前または直後に以下の共通注記を必ず配置します。
  > [!NOTE]
  > 本件に関連した、公開されているYouTube動画のURLを参考までに掲載します。リンク切れがあった場合、お手数ですが本ワークブック管理者までお知らせください。
- **挙動**: 外部リンクとしての別タブ挙動等は、システム側で自動適用されます。
