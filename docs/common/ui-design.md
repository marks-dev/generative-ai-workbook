# UI Design

## 目的

本ファイルはUI設計方針を管理する。

レイアウト詳細やコンポーネント詳細ではなく、サイト全体のUI設計思想を記録する。

---

## 対象デバイス

優先対象:

- デスクトップPC
- ノートPC

対応対象:

- タブレット
- スマートフォン

---

## 基本方針

- PCファーストで設計する
- レスポンシブ対応を行う
- スマートフォンでの閲覧を可能とする
- スマートフォンでの学習も考慮する
- ただし全コンテンツのスマートフォン完遂は前提としない

例:

- AIコーディング
- MCP
- RAG

などはPC利用を前提としてもよい。

---

## デザイン方針

参考イメージ:

- GitBook系の情報設計
- Notion系の読みやすさ
- サントリーサイトの柔らかい雰囲気

方針:

- 読みやすさを最優先とする
- 派手さより理解しやすさを重視する
- 学習継続しやすいUIを目指す
- 長文コンテンツの可読性を重視する
- カテゴリページはカードUIを活用する

配色方針:

- ライトテーマ（明るく落ち着いたトーン）を採用します（具体的な定義は design-token-design.md に一任）。
- 画面全体の眩しさを抑え、長文の可読性を高めるため、背景には純白ではなく少し柔らかい背景色を利用します。
- 強いブランドカラーへの依存を避け、学習の邪魔にならないトーンを優先します。

避けるもの:

- 過度なアニメーションや装飾過多なUI
- 強すぎるグラデーション
- ダークテーマ前提の設計（サイト全体およびスライド等の教材アセットも対象外）

---

## 共通レイアウト構造（スケルトン）

サイト共通の基本骨格として、HeaderおよびMain Contentを配置する縦2段構造を基本とします。

```text
┌──────────────────────────────┐
│ Header                       │
│ ├ Global Area                │
│ └ Context Area               │
├──────────────────────────────┤
│ Main Content                 │
└──────────────────────────────┘
```

### 画面別レイアウトの拡張許容

* **基本ルール**: サイト全体の整合性を保つため、最上位のHTML構造は上記の「Header + Main Content」を共通とします。
* **画面別アレンジ**: `Main Content` 内部のレイアウト（サイドバーの有無、2カラム化、アコーディオンメニューの追加など）については、各個別画面設計書（`docs/pages/*.md`）の定義に従って、柔軟に拡張・定義することを許容します。
