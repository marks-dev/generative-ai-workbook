# UI Design (サイト共通UI設計方針)

## 目的
本ファイルは、サイト全体のUI設計思想、共通の表示/インタラクションルール、および共通レイアウト構造を定義・管理します。

---

## 対象デバイスと基本方針
- **優先対象**: デスクトップPC、ノートPC
- **対応対象**: タブレット、スマートフォン（※スマートフォンでの学習も考慮しますが、全コンテンツのスマートフォン完遂は前提としません。PC利用を推奨します。）
- **可読性優先**: GitBook系の情報設計、Notion系の読みやすさを目指し、まぶしさを抑えたライトテーマの柔らかい背景色を採用します。

---

## 共通レイアウト構造（スケルトン）
サイト共通の基本骨格として、HeaderおよびMain Contentを配置する縦2段構造を基本とします。詳細は [header.md](../components/header.md) および各個別画面設計書（`docs/pages/*.md`）を参照。

---

## 外部リンクの取り扱い (共通ルール)
現在のドメインと異なる外部サイトへのリンク（MIT Licenseや参考ドキュメントへのリンクなど）の挙動方針を定義します。

- **方針**: 外部リンクは、現在の学習セッション（コンテキスト）を維持するため、常にブラウザの別タブで開きます（`target="_blank"`）。
- **セキュリティ**: セキュリティ保護のため、必ず `rel="noopener noreferrer"` を付与します。
- **自動化実装**: 共通の JavaScript (`learning-progress.js` 等) にて、ページロード時に `http` または `https` から始まるすべてのリンクを自動走査し、現在のホスト名と異なる場合にこれらの属性を付与します。これにより、手動での属性付与漏れを防ぎ一貫性を保証します。

---

## コンテンツ装飾系UI仕様 (Markdown拡張)
Markdownからレンダリングされる要素、またはカスタムHTML要素に適用される共通の装飾UI設計です。

### 1. Alert / Tip (Notion風のアラートボックス)
- **Alert**: 注意事項やハルシネーション警告を表示（`[!IMPORTANT]` / `[!WARNING]` 構文から変換）。強い警告色ではなく、Notion風の背景とアイコンで表現。
- **Tip / Note**: 補足知識や実務ノウハウを表示（`[!TIP]` / `[!NOTE]` 構文から変換）。
- **共通**: 視覚的な囲み枠、十分なパディング、および左端の識別用アイコンでレイアウトを際立たせます。

### 2. Exercise (理解確認・実践課題)
- ハンズオンや練習問題として、通常の本文やAlertと区別できる専用の囲み枠、タイポグラフィ（タイトル＋タスク指示）を定義します。

### 3. Code Block (等幅コード表示)
- 動的コピーボタン: JavaScriptにより、全コードブロックの右上に動的に「📋 コピー」ボタンを配備します。
- レイアウト配慮: ボタンとソースコードのテキストが重ならないよう、コードブロック上部に適切な余白（padding-top）を確保する共通スタイルを適用します。

### 4. Table (データ比較)
- 罫線の太さを極力抑え、交互の行の背景色（縞模様）やヘッダーの強調スタイルなど、Notion風のシンプルで視認性の高いテーブルデザインを適用します。
