

# Design Token Design

## 目的

デザインで利用する共通トークンを定義する。

実装時の見た目の揺れを防止することを目的とする。

---

## 基本方針

- デザイン値を共通化する
- CSSへ直接値を書かない
- トークン経由で利用する
- AI実装時の差異を最小化する
- 長期保守しやすい構成とする

---

## 位置付け

```text
Design Token
↓
site.css
↓
各画面
```

補足:

- 実装時はDesign Tokenを利用する
- 個別値の乱立を防止する

## Color Token

用途と具体的な定義値:

| トークン名 | 実体値 | 用途説明 |
| :--- | :--- | :--- |
| `--color-background` | `#f7f5f0` | サイト全体の基本背景色（純白を避けた柔らかい砂白色）。 |
| `--color-surface` | `#ffffff` | カードやダイアログなどの表面背景色。 |
| `--color-surface-muted` | `#fbfaf7` | 補助的なセクション背景色。 |
| `--color-text` | `#242424` | 基本のテキスト文字色。 |
| `--color-text-muted` | `#66645f` | 補助テキスト（補足やメタ情報）の文字色。 |
| `--color-border` | `#ddd8cc` | カードの枠線や区切り線の境界線色。 |
| `--color-accent` | `#2f6f73` | プライマリーアクセント色（落ち着いた青緑色・ティール）。 |
| `--color-accent-muted` | `#e6f0ef` | アクセントの背景用の極薄ティール色（完了カード背景など）。 |
| `--color-focus` | `#1f5f91` | キーボードフォーカス時の輪郭色。 |

## Typography Token

用途と具体的な定義値:

| トークン名 | 実体値 | 用途説明 |
| :--- | :--- | :--- |
| `--font-family-base` | `system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` | サイト全体の基本フォントファミリー。 |
| `--font-size-small` | `0.875rem` | 補助テキストやメタ情報の文字サイズ。 |
| `--font-size-body` | `1rem` | 基本本文の文字サイズ。 |
| `--font-size-h3` | `1.25rem` | 小見出し（H3）の文字サイズ。 |
| `--font-size-h2` | `1.625rem` | 中見出し（H2）の文字サイズ。 |
| `--font-size-h1` | `2.25rem` | 大見出し（H1）の文字サイズ。 |
| `--line-height-body` | `1.8` | 本文の行高（読みやすさを考慮した広めの設計）。 |
| `--line-height-heading` | `1.35` | 見出しの行高。 |

#### インライン装飾値
- **太字 (`strong`) のフォントウェイト**: `700`
- **インラインコード (`code`) のフォントサイズ**: `0.9em`

## Spacing Token

用途と具体的な定義値:

| トークン名 | 実体値 | 用途説明 |
| :--- | :--- | :--- |
| `--space-xs` | `0.5rem` | 極小余白（インライン間隔やリスト間隔等）。 |
| `--space-sm` | `0.75rem` | 小余白（ヘッダー要素の間隔やカードパディング等）。 |
| `--space-md` | `1rem` | 中余白（標準的なマージン・パディング）。 |
| `--space-lg` | `1.5rem` | 大余白（コンテナ周囲の余白や見出し周囲等）。 |
| `--space-xl` | `2.5rem` | 特大余白（大項目同士の間隔等）。 |
| `--space-2xl` | `4rem` | 最大余白（コンテンツ上下余白等）。 |

## Border Token

用途と具体的な定義値:

| トークン名 | 実体値 | 用途説明 |
| :--- | :--- | :--- |
| `--border-width` | `1px` | 基本的な境界線の太さ。 |

#### 見出しアクセント線の太さ
- **H1 アクセント下線の太さ**: `2px`
- **H2 アクセント縦バーの太さ**: `6px`
- **H3 左ボーダーの太さ**: `4px`

## Layout Container Token

用途と具体的な定義値:

| トークン名 | 実体値 | 用途説明 |
| :--- | :--- | :--- |
| `--container-width` | `1040px` | サイト全体および共通ページのコンテンツ最大幅。 |
| `--content-width` | `760px` | 旧レイアウト等における記事本文幅の基本値。 |

---

## Responsive Token

用途と具体的な定義値:

| 基準値 | 実体値 | 用途説明 |
| :--- | :--- | :--- |
| **Mobile / PC 境界しきい値** | `720px` / `721px` | サイドバーのDrawer（アコーディオン）トグル表示と固定表示の境界。 |
| **サイドバー衝突回避しきい値** | `1120px` | フローティングサイドバーと本文テキストの重なりを防止するメディアクエリしきい値。 |

---

## 利用ルール

推奨:

- トークン経由で値を参照する

非推奨:

- 個別コンポーネントで独自値を定義する

目的:

- 一貫性維持
- 保守性向上
