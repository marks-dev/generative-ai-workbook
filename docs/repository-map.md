

# Repository Map

## 目的

このファイルは、リポジトリ全体の構造を人間およびAIが素早く把握するための索引である。

実際のディレクトリツリーを記録することが目的ではない。

以下を管理する。

- どのファイルが何を管理しているか
- どの変更を行う時にどのファイルを参照すべきか
- ファイル間の責務の境界

---

## 設計ドキュメント

### docs/site-map.md

サイト構造と学習導線を管理する。

変更例:

- カテゴリ追加
- 教材構成変更
- 学習導線変更

### docs/url-design.md

URL設計を管理する。

変更例:

- URL追加
- URL命名規則変更
- ディレクトリ公開構造変更

### docs/layout-design.md

画面レイアウト設計を管理する。

変更例:

- ヘッダー
- フッター
- サイドバー
- ナビゲーション

### docs/component-design.md

共通コンポーネント設計を管理する。

変更例:

- ボタン
- カード
- アラート
- コードブロック

### docs/ui-design.md

デザイン方針を管理する。

変更例:

- DESIGN.md選定
- 配色
- 余白
- タイポグラフィ

### docs/repository-map.md

本ファイル。

リポジトリ全体の索引を管理する。
