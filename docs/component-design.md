# Component Design

## 目的

本ファイルは共通コンポーネント設計を管理する。

各画面で再利用するUI部品を定義し、デザインおよび実装の統一を行う。

---

## 基本方針

- 共通化できるものは共通化する
- 学習体験の一貫性を重視する
- 過度なコンポーネント分割は行わない
- 読みやすさを最優先とする

---

## コンテンツ系

### Category Overview

用途:

- カテゴリ説明表示
- 学習目安表示
- 推奨対象表示

利用画面:

- カテゴリページ

補足:

- Heroコンポーネントは採用しない
- 大きなビジュアル領域は作らない
- シンプルな概要表示を優先する

### Learning Card

用途:

- 教材一覧表示
- 教材概要表示

利用画面:

- カテゴリページ

表示内容:

- アイコン
- タイトル
- 概要
- 難易度
- 推奨プラン
- 利用サービス
- 完了状態

補足:

- 一覧性を優先する
- レイアウト図は概念図とする

### Navigation Tile

用途:

- 学習ルート入口表示
- 主要カテゴリ入口表示

利用画面:

- トップページ
- 必要に応じてカテゴリ入口画面

表示内容:

- 利用者が遷移先を識別できる視覚要素
- タイトル

方針:

- カード全体をリンクとして扱う
- 内部に別ボタンを配置しない
- 説明文は原則として持たない
- 入口であること、押せることを視覚的に分かりやすくする
- hover / focus-visible で操作可能であることを示す
- 外部アイコンライブラリは必要になるまで追加しない

補足:

- 詳細説明はタイル内ではなく、タイルの前にある説明セクションで行う
- 情報量を抑え、学習導線を迷わせないことを優先する

### Related Content Card

用途:

- 関連教材表示

利用画面:

- 教材ページ
