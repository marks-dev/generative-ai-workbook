# Top Page Design

## 目的

トップページは本サイトの入口とする。

利用者が自分に合った学習ルートを選択し、学習を開始または再開できることを目的とする。

---

## 基本方針

- Heroは採用しない
- 学習導線を最優先とする
- 「読んで学ぶ」と「触って学ぶ」を同列に扱う
- 初回利用者と継続利用者の両方を支援する

---

## 構成

```text
Header

┌─────────────────┬─────────────────┐
│ 読んで学ぶ      │ 触って学ぶ      │
└─────────────────┴─────────────────┘

学習の進め方
```

---

## 学習の進め方

表示内容:

- 本サイトの利用方法
- 読む派・触る派の学習スタイル説明
- 学習状況保存方法の説明

補足:

- トップページ内で完結することを優先する
- LocalStorage利用について説明する
- JSONエクスポート・インポートについて説明する

---

## 学習ルート入口

目的:

- 利用者が学習ルートを直感的に選択できるようにする
- 「読んで学ぶ」と「触って学ぶ」を同列の入口として扱う

表示方針:

- 説明カードではなく、ナビゲーションタイルとして表示する
- カード全体を遷移部品とする
- カード内に個別のボタンは配置しない
- カード内の情報量は最小限にする
- どちらを選ぶべきかの説明は「学習の進め方」で行う
- トップページは情報過多を避け、利用者が主要な学習導線を素早く把握できる構成を優先する

表示内容:

- 学習ルートを識別できる視覚要素
- ルート名

遷移先:

- 読んで学ぶ: 読んで学ぶトップ
- 触って学ぶ: 触って学ぶトップ

レスポンシブ方針:

- PCでは2つのタイルを横並びにする
- スマートフォンでは縦並びにする
- 横スクロールが発生しない構成とする
