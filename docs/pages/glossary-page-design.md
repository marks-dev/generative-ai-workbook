# Glossary Page Design

## 目的

用語集ページは、生成AIの学習や自習式ワークブック内でよく使われる専門用語を解説する画面です。
利用者が教材を読み進める中で不明な言葉に遭遇した際、即座に意味を調べられるように支援することを目的とします。

---

## 基本方針

- Notion風のシンプルで無駄のないデータベース風のテーブルUIを採用する。
- JavaScriptによる高速なインクリメンタル検索（リアルタイム絞り込み）を提供する。
- 検索用のタグ（読み方や類義語など）をあらかじめ定義し、表記の揺れやひらがな・カタカナ入力でも検索可能にする。
- サーバー連携を行わず、フロントエンド単体（HTML/CSS/JS）で完結させる。

---

## 各セクションの詳細

### 1. 検索入力エリア
- **要素**: `input[type="text"]`
- **識別属性**: `data-glossary-search`
- **デザイン**: 画面幅いっぱいに広がるモダンな角丸テキストボックス。プレースホルダーで入力例を表示。

### 2. 用語一覧テーブル
- **要素**: `table.glossary-table`
- **構成**:
  - `thead`: 「用語」と「解説」のヘッダー（`th`）。
  - `tbody`: 用語アイテムの一覧。各行（`tr`）は `class="glossary-item"` を持ち、かつ検索用インデックスとなる `data-search-terms` 属性を持つ。
- **データ形式例**:
  ```html
  <tr class="glossary-item" data-search-terms="rag 検索拡張生成 けんさくかくちょうせいせい 社内データ 外部検索">
    <td class="glossary-table__term"><strong>RAG (Retrieval-Augmented Generation / 検索拡張生成)</strong></td>
    <td class="glossary-table__def">AIが質問に回答する際、社内文書やインターネット上の最新情報などを「検索」し...</td>
  </tr>
  ```

### 3. 結果なしメッセージ
- **要素**: `p.glossary-no-results`
- **初期状態**: 非表示（`hidden` 属性）
- **挙動**: 検索にヒットする項目がゼロになった際、テーブル全体を非表示にしてこのメッセージを表示する。

---

## 検索ロジック仕様（リアルタイム検索）

JavaScript (`learning-progress.js`) の `applyGlossarySearch` 関数で制御します。

1. **入力監視**:
   - `data-glossary-search` インプットの `input` イベントをトリガーにする。
2. **比較判定**:
   - 入力値（小文字変換、両端トリム）を取得。
   - 各 `glossary-item` について、以下の3つのテキストとの部分一致（`indexOf`）を走査する：
     - 用語タイトル (`.glossary-table__term` のテキスト）
     - 用語解説 (`.glossary-table__def` のテキスト）
     - 属性値 `data-search-terms` の内容
3. **表示切り替え**:
   - 部分一致した項目は `hidden = false`、不一致の項目は `hidden = true` に設定。
   - 一致する項目が1つでもある場合：テーブルラッパー (`.glossary-table-wrapper`) を表示、結果なしメッセージ (`.glossary-no-results`) を非表示。
   - 一致する項目が0の場合：テーブルラッパーを非表示、結果なしメッセージを表示。

## レスポンシブ・スタイル方針

- 共通のコンテンツページレイアウト（最大幅、パディング等）については、[ui-design.md](../common/ui-design.md) に定義された共通レイアウトおよび [css-design.md](../common/css-design.md) のレイアウト方針に従います。
- 用語解説テーブル独自の表示崩れ対策として、スマートフォンなど幅の狭い画面では、CSSで適切な最小幅（`min-width`）の確保とフォントサイズ調整を行い視認性を維持するとともに、テーブルを包むコンテナに `overflow-x: auto` を適用して画面外へのはみ出しを防ぎます。

---

## 用語集の更新ルール

- 個別に「用語集の最新化」の指示があった場合のみ、作成済みの教材本文（`content/` または `src/` 配下の markdown ファイル）から用語を棚卸しし、既存の用語集に未登録の用語のみを抽出して新たに追加します。
- 日常の教材追加・更新時における都度の同期・付き合わせ運用は行いません。
