# Category Page Design

## 目的

カテゴリページは学習カテゴリの入口とする。

利用者がカテゴリの概要を理解し、学習したい教材を選択できることを目的とする。

---

## 基本方針

- カテゴリ概要（詳細説明や推奨対象・目安等のメタデータ）は表示しない
- 教材一覧を主役とする
- Heroおよび導入テキストは採用しない
- 一覧性を重視したリスト型カードを採用する
- `h1` はスクリーンリーダー用に `visually-hidden` でマークアップし、画面上は `h2` の教材一覧から直接開始する

---

## 構成

```text
Header
├ Global Area
└ Context Area
  └ Breadcrumb

h1 (visually-hidden)

教材グループ見出し (h2)
Learning Card
Learning Card
Learning Card
...
```

---

## Breadcrumb

目的:

- 現在位置表示
- 上位階層への移動

例:

```text
読んで学ぶ > 基礎
```

---

## Category Overview

表示方針:

- 画面上には一切表示しない。
- ただし、アクセシビリティ（スクリーンリーダー等）確保のため、マークアップ上は `h1` を `visually-hidden` クラスで保持する。

---

## 教材一覧

目的:

- 学習教材選択

表示方法:

- Learning Cardを利用する
- アイコン＋説明文の極めてシンプルなテキストレイアウトを採用する

表示内容:

- 教材タイトル
- 教材概要（説明文）
- 完了状態（右側中央の `✅` バッジ）

補足:

- カテゴリ内の教材を一覧表示する
- 完了済み教材はカード全体に subtle なグリーンのボーダーと淡い背景（`.is-completed`）を付与し、右端中央に `✅` バッジを表示して視覚的に区別する。
- `h2` でレベル別などのグルーピングを直接表示する。

---

## Learning Card

レイアウト:

```text
┌────┬─────────────────────────────────┬─────┐
│Icon│ タイトル                        │ [✅]│
│    │ 概要説明                        │     │
└────┴─────────────────────────────────┴─────┘
```

表示内容:

- アイコン
- タイトル
- 概要（説明文）
- 完了状態（`✅` バッジ。完了時のみ右端中央に表示）

動作:

- カード全体がリンクとなっており、クリック/タップで教材ページへ遷移する。

補足:

- カード全体を3カラムのグリッド/フレックスボックスレイアウト（左: アイコン、中: タイトルと説明文、右: 完了バッジ）で構成する。
- 完了状態（`.is-completed`）の時は、カードの境界線が subtle なグリーン（`#2e7d32`等）になり、背景色が非常に淡いグリーン（`#e8f5e9`等）へトグルされる。
- 親カード（ポータルカテゴリカード）は、配下の子教材がすべて完了している時のみ完了バッジ `✅` が表示され、カード全体が完了状態へトグルされる。

---

## 教材メタ情報

各教材は以下のメタ情報を管理・保持する。

- タイトル (`title`)
- 概要 (`description`)
- アイコン (`icon`)
- 完了判定用ID (`completionId`) - 詳細教材に定義
- 完了判定用ID配列 (`completionIds`) - 親カテゴリのポータルカード等に定義

補足:

- 以前設計されていた難易度、推奨プラン、利用サービスなどのメタバッジ群は、ページの簡素化と視認性向上のため、表示上は廃止された。
- ローカルストレージ内の `completionId`（または親カードの `completionIds` に含まれるすべてのID）の完了状態を検知して、フロントエンド JavaScript が動的に完了状態を反映する。

---

## レベル別グルーピング

目的:

- 学習順序を明確にする
- 同一カテゴリ内で教材を階層的または難易度・ステップ順にまとめる

表示例:

```text
基礎
──────────
📘 生成AIとは？
📘 生成AIの得意・不得意

チャットレベル
──────────
📘 プロンプトエンジニアリング基礎
📘 指示書の書き方
```
