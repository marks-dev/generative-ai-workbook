# Content Page Design (教材詳細ページ設計)

## 目的
教材ページは学習コンテンツを提供する画面です。利用者が教材を読み進め、理解し、次の教材へ進めることを目的とします。

---

## 基本方針
- **可読性最優先**: 長文の教材コンテンツを読みやすくするため、マージンや余白、文字の読みやすさに配慮します。
- **教材本文への集中**: Previous/Next ナビゲーションや不要なメタバッジ（難易度等）を排し、シンプルなデザインを維持します。
- **マルチデバイス対応**: PCではサイドバーを展開し、スマートフォンではフローティング形式で学習を支援します。

---

## 画面レイアウト構成

### 1. PCレイアウト (画面幅 721px 以上)
- 共通Header（固定）を最上部に配置。
- 左側に `Sidebar`、右側に `Content`（メインコンテンツ）を配置する2カラムレイアウトを基本とします。
- 本文エリア（`.content-article`）は最大幅 `1040px` (`var(--container-width)`) まで拡張された中央寄せのレイアウトです。

### 2. スマートフォンレイアウト (画面幅 720px 以下)
- 共通Header（固定）を最上部に配置。
- `Sidebar` は画面右下のフローティングボタン（`☰` / `✕`）によりポップオーバー開閉する方式となります。
- `Content` は左右対称の均等な余白で画面いっぱいに広がる単一カラムとなります。

---

## 構成コンポーネント (参照)
教材詳細ページを構成する各UI部品の具体的なタグ構造、クラス名、および詳細な挙動は、以下の各コンポーネント設計書を参照してください。

- **[Header (ヘッダー)](../components/header.md)**: ロゴ、グローバルナビゲーション、パンくず、完了ボタンのレイアウト配置。
- **[Sidebar (サイドバー)](../components/sidebar.md)**: カテゴリ内教材一覧、進捗同期、およびスマートフォン表示時のフローティングアコーディオン/バックドロップ制御。
- **[Breadcrumb (パンくずリスト)](../components/breadcrumb.md)**: ヘッダー内の現在地階層表示。
- **[Complete Button (完了ボタン)](../components/complete-button.md)**: 学習完了トグル動作とLocalStorage同期。
- **[Completion Prompt (完了誘導)](../components/completion-prompt.md)**: 未完了状態でスクロールが末尾に到達した際にヘッダーの完了ボタン直下に出現する吹き出し（グラスモーフィズム、3秒自動消滅）。
- **[Related Content Card (関連教材)](../components/related-content-card.md)**: 本文最下部のまとめカード内に配置される、座学とハンズオンを繋ぐ動的リンクカード。
- **[Content Header (教材ヘッダー)](../components/content-header.md)**: メインコンテンツ最上部のタイトル・概要説明のレイアウト表示。
- **スクロールナビゲーション (↑・↓ボタン)**: 教材詳細ページのセクション（ヘッダー、準備、各ステップ、まとめカード）間をスムーズにアンカースクロール移動するためのUI（仕様は [Sidebar (サイドバー)](../components/sidebar.md) に集約）。

## コンテンツエリアのスタイル定義
Markdown レンダラーから生成される各要素（見出し、Alert, Tip, Code Block, Table 等）の共通UI仕様、および外部リンクのセキュリティ/別タブ挙動、執筆者向けのMarkdown記述ガイドラインやメディアアセット掲載ルールについては、**[ui-design.md](../common/ui-design.md)** を参照してください。
