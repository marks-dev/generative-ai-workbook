# Content Common Design

## 目的と基本方針
本ファイルは、教材コンテンツ全体の共通管理方式、記述ルール、および共通のメディアアセット掲載基準を定義し、AI生成時や手動執筆時の品質および一貫性を安定させることを目的とします。

- **Markdown正本主義**: 原則としてコンテンツはMarkdownで管理し、HTMLは生成物とします。人間が直接修正しやすい構成、およびAIが生成しやすい構成を優先します。
- **標準Markdown優先**: 標準Markdownを中心として運用し、独自記法は必要になるまで追加しません。

---

## コンテンツ管理方式とフロー
教材コンテンツのビルドおよびデプロイは以下の静的生成フローに従います。
```text
Markdown (正本)
↓
HTML生成 (Eleventy)
↓
GitHub Pages公開 (GitHub Actions)
```
- **編集方針**: 人間およびAIはMarkdownファイルのみを編集対象とし、HTMLを直接編集することは禁止します。
- **管理対象**: 教材本文、カテゴリ情報、用語集、このサイトについて。

---

## 共通の記述ルール
Markdown本文を記述する際は、以下の要素別ルールを厳守してください。

- **見出し階層**: `## 大項目` ＞ `### 中項目` ＞ `#### 小項目` の最大4階層とし、深くしすぎない。
- **Alert / Tip**:
  - 注意事項やハルシネーション警告には `[!IMPORTANT]` / `[!WARNING]` を使用。
  - 補足やノウハウには `[!TIP]` / `[!NOTE]` を使用。
- **Exercise**: 理解確認や実践課題を掲載（解答は別教材にする等の運用を許容）。
- **Code Block**: コピーしやすさを重視。全コードブロックに「📋 コピー」ボタンをJS自動生成し、ボタンと文字の重なりを防ぐCSS余白（padding-top）を確保する。
- **Table**: データの比較や一覧表示において、表形式が明らかに分かりやすい場合のみ利用。
- **独自記法方針**: 専用の独自記法（Alert, Tip, Exercise等）は慎重に導入し、基本的には標準的なMarkdown表現、またはシステム側で共通変換可能な記法（GitHub-style Alertsなど）に準拠する。

---

## メディア掲載共通ガイドライン
教材ページ内（メインコンテンツ）に画像や動画を掲載する場合は、以下のルールを厳密に遵守します。

### 1. 画像アセット (WebP形式に完全統一)
- **原則と配置**: ビジュアル補足が有効な場合は積極的に掲載。画像は**WebP形式（`.webp`）のみ**とし、`src/assets/images/` 配下に格納し相対パスで呼び出します。
- **AI生成動作**: AIが自動生成する際、ライセンスフリーの画像が利用可能な場合は、取得・WebP変換・格納・Markdown埋め込みを自動で実行します。

### 2. YouTube動画 (埋め込み禁止・注記付き外部リンク化)
- **原則**: 直接の iframe 埋め込みは一切禁止（将来のリンク切れやCORS警告を防止）。
- **掲載方式**: 必ず別タブで開く外部リンクとし、直前または直後に以下の共通注記を必ず配置します。
  > [!NOTE]
  > 本件に関連した、公開されているYouTube動画のURLを参考までに掲載します。リンク切れがあった場合、お手数ですが本ワークブック管理者までお知らせください。

## AI生成時のルール
AIがコンテンツを自動生成する際は、以下のルールに従います。
- **推奨**: 初心者を前提にした平易な日本語、用語の初出時解説、小ステップ構成、箇条書きの活用、結論先行、太字（`**`）は本当に重要なキーワードのみに限定。
- **禁止**: 不要な前置き、長文の囲みなどの過剰な太字装飾、冗長な説明、**既存教材コンテンツの参照・模倣・記述の寄せ（合わせ）行為**（※常に本仕様書および各系統仕様書のみを正として新規生成し、再現性を担保する）。
- **日本人学習者への配慮**: 掲載・紹介するメディアアセットは、学習者が混乱しないよう、**可能な限り日本語で作成・説明されているもの**を採用します。
