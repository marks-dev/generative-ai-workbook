# Content Common Design (教材コンテンツ共通執筆ルール)

## 目的と基本方針
本ファイルは、教材コンテンツ全体の共通記述ルール、およびメディアアセット掲載基準を定義し、人間による執筆およびAIによる自動生成時のコンテンツ品質と執筆ルールの整合性を保つことを目的とします。
- **Markdown正本主義**: 原則としてコンテンツはMarkdownで管理し、HTMLは生成物とします。人間が直接修正しやすく、AIが生成しやすい構成を優先します。
- **標準Markdown優先**: 標準Markdownを中心として運用し、独自記法は極力導入しません。

---

## 共通の記述ルール (執筆者向け)
Markdown本文を記述・生成する際は、以下のルールを厳守してください。

- **見出し階層**: `## 大項目` ＞ `### 中項目` ＞ `#### 小項目` の最大4階層とし、深くしすぎない。
- **外部リンクの記述方式**:
  - 執筆時（Markdown上）は、標準のMarkdownリンク記法 `[アンカーテキスト](URL)` で記述します。
  - セキュリティ属性（`rel`）や別タブ表示（`target="_blank"`）の指定は、システム側（共通JS）がロード時に自動付与するため、Markdown内にHTMLタグなどを手動で記述する必要はありません（詳細は [ui-design.md](../common/ui-design.md) を参照）。
- **Alert / Tip**:
  - 注意事項やハルシネーション警告には `[!IMPORTANT]` / `[!WARNING]` 構文を使用します。
  - 補足やノウハウには `[!TIP]` / `[!NOTE]` 構文を使用します。
  - 具体的な背景色や装飾などのUI定義は [ui-design.md](../common/ui-design.md) を参照。
- **Exercise (実践課題)**:
  - 理解確認や実践課題を掲載する際は、専用の章または記述ブロックとして定義します。
- **Code Block (コードブロック)**:
  - ソースコードやコマンドは、標準のバッククォーツ3つで囲み、必ず言語名を指定します（「📋 コピー」ボタンおよび余白スタイルはシステムが自動処理するため、Markdown上は標準ブロックとして記述します）。
- **Table (テーブル)**:
  - データの比較や一覧表示において、表形式が明らかに分かりやすい場合のみ利用します。

## メディア掲載ガイドライン (末端画面用)
教材ページ内（メインコンテンツ）に画像や動画を掲載する場合は、以下のルールを遵守します。

### 1. 画像アセット (WebP形式に完全統一)
- **配置**: 画像は**WebP形式（`.webp`）のみ**とし、`src/assets/images/` 配下に格納し相対パスで呼び出します。
- **AI生成動作**: AIが自動生成する際、ライセンスフリーの画像が利用可能な場合は、取得・WebP変換・格納・Markdown埋め込みを自動で実行します。

### 2. YouTube動画 (埋め込み禁止・注記付き外部リンク化)
- **原則**: 直接の iframe 埋め込みは一切禁止（将来のリンク切れやCORS警告を防止）。
- **掲載方式**: 必ず標準のリンクとして記述し、直前または直後に以下の共通注記を必ず配置します。
  > [!NOTE]
  > 本件に関連した、公開されているYouTube動画のURLを参考までに掲載します。リンク切れがあった場合、お手数ですが本ワークブック管理者までお知らせください。
- **挙動**: 外部リンクとしての別タブ挙動等は、システム側で自動適用されます。
