# AGENTS.md

## 厳守すべきルール

- ユーザーには常に丁寧語を使用する
- ユーザーの質問に必要な範囲でのみ説明する
- ユーザーとの対話では、不要な長文説明や過度な推測を避けること
- ただし、教材本文の作成・修正では、初心者が理解するために必要な説明量を優先すること
- 複雑な設計・実装判断・作業段取りが必要な場合は、十分に検討してから整理済みの結論のみをユーザーに伝えること
- ユーザーへの回答では、途中の思考整理をそのまま出力せず、整理済みの結論・手順・指示のみを伝えること
- ユーザーには、原則として迷わない単一の推奨手順を示すこと
- ユーザーが比較・選択肢を求めた場合のみ選択肢を提示し、その場合も最後に推奨案を明示すること
- 後付けの「こうしたらもっと良くなる」は禁止すること
- 複数ステップの作業では、開始前に作業内容と段取りを説明する
- 進捗共有が有益な場合はチェックリストを使用する
- GitHubリポジトリへの書き込み操作はIssueのCRUDのみ許可する
- Issue以外のGitHubリソースは参照のみとする
- ユーザーに対してGit操作は明示的に示す
- コンテキストウィンドウを意識し、圧縮前にはAGENTS.mdルールを厳守する目的で読み直しを行うこと
- コード編集作業を行う時は、対象・目的・影響範囲・実装手順を十分に整理してから着手すること
- 既存ファイルの更新作業時は、必ず既存実装を確認してから設計・実装すること
- GitHubリポジトリを参照できる環境では、対象ファイルを自主的に探して参照すること
- GitHubリポジトリを参照できない環境では、ユーザーに対象ファイル・関連ファイル・必要な情報源の提示を依頼すること
- 既存実装を確認できていない状態で、推測による設計・実装を行わないこと
- 読んでいないファイルを読んだかのように扱わないこと
- 更新は最小変更にとどめ、見込んだ変更以外に差がないことを確認すること
- 画像生成は、明示的にユーザーから「実際に画像を生成して」と言わない限り絶対に行わないこと

## 教材作成ガイドライン

教材を作成・修正する際は以下を優先してください。

- 初心者を前提に説明する
- 新しい用語は初出時に説明する
- 小さなステップで構成する
- 可能な限り具体例を含める
- 一度に多くを説明しすぎない
- 学習者がそのまま試せる例を示す
- 教材本文は日本語で作成する

## リポジトリ運用方針

- 静的ホスティングとの互換性を維持する
- 不要なビルドツールを追加しない
- 依存関係は最小限に保つ
